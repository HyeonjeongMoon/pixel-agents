## 🔴 긴급 — 즉시 수정

### 백엔드

**B-1. Race condition: `layoutPersistence.ts`의 `markOwnWrite()`**
- `lastMtime` 업데이트 타이밍 문제 — 플래그를 `stat()` 이후에 설정해 경쟁 조건 발생
- 수정: `skipNextChange = true`를 `stat()` 이전에 먼저 설정

**B-2. 캐릭터 스프라이트 부분 실패 처리 누락 (`assetLoader.ts:341-397`)**
- 6개 파일 중 하나라도 누락 시 전체 로딩 실패 → webview에 캐릭터 없음
- 수정: 개별 파일 실패를 try/catch로 처리, 성공한 파일만 전송

**B-3. `setTimeout` 콜백이 삭제된 agent의 webview로 메시지 전송 (`transcriptParser.ts:125-131`)**
- 300ms 지연 중 agent 삭제 시 dangling reference
- 수정: 타임아웃 발동 시점에 agentId 유효성 재확인

**B-4. `fileWatcher.ts`에서 파일 크기 축소 케이스 미처리**
- `stat.size <= agent.fileOffset` 체크 이후 `fs.readSync` 전 파일 재확인 없음
- Windows에서 파일 락 시 예외 발생 가능
- 수정: try/catch 추가 + 오프셋 리셋 로직

**B-5. `fs.watch` 에러 핸들러 부재 (`fileWatcher.ts`)**
- `fs.watch` 에러 발생 시 폴링으로 자동 페일오버하는 로직 없음
- 수정: `'error'` 이벤트 리스너 추가 → 폴링 모드로 전환

**F-5. `OfficeCanvas.tsx:73-74` — `imageSmoothingEnabled` 중복 설정**
